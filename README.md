## 🚀 ИНСТРУКЦИЯ ПО ЗАПУСКУ

### 1. Установка зависимостей:
```bash
npm install
```

### 2. Настройка базы данных MySQL:
```bash
# Создайте базу данных
node setup-db.js
```

### 3. Запуск сервера:
```bash
npm start
# или для разработки с автоперезагрузкой:
npm run dev
```

### 4. Откройте в браузере:
```
http://localhost:3000
```

## 📝 ПРОМТ ДЛЯ СОЗДАНИЯ ПРОЕКТА

```
Создай современный веб-мессенджер с использованием Node.js, Express, WebSocket, MySQL и vanilla JavaScript.

Требования к функционалу:

1. Аутентификация и пользователи:
   - Регистрация с username, email, password
   - Вход по username/email и password
   - JWT токен для авторизации
   - Редактирование профиля (email, пароль)
   - Загрузка и обрезка аватара (Cropper.js)
   - Отображение статуса online/offline

2. Чаты и сообщения:
   - Создание личных чатов между пользователями
   - Отправка сообщений в реальном времени (WebSocket)
   - Отправка файлов и изображений
   - Отметка о прочтении (галочки)
   - Индикатор набора текста (печатает...)
   - Удаление чата (скрытие истории для пользователя)
   - Очистка истории сообщений
   - Автоматическое восстановление чата при новом сообщении

3. Интерфейс:
   - Адаптивный дизайн
   - Светлая и темная тема
   - Поиск по чатам и контактам
   - Браузерные уведомления
   - Счетчик непрочитанных сообщений
   - Модальные окна для профиля и обрезки аватара

4. База данных (MySQL):
   - Таблицы: users, chats, chat_participants, messages, deleted_chats, deleted_messages
   - При удалении чата сообщения помечаются как удаленные только для этого пользователя
   - Другой участник продолжает видеть всю историю

5. API Endpoints:
   POST   /api/auth/register
   POST   /api/auth/login
   GET    /api/auth/me
   GET    /api/users
   GET    /api/chats
   POST   /api/chats
   DELETE /api/chats/:id
   DELETE /api/chats/:id/clear
   GET    /api/chats/:id/messages
   POST   /api/upload
   PUT    /api/users/profile
   PUT    /api/users/password
   POST   /api/users/avatar

6. WebSocket события:
   Клиент → Сервер: auth, message, typing
   Сервер → Клиент: auth_success, message, typing, status

Технологии:
- Бэкенд: Express.js, WebSocket (ws), JWT, bcryptjs, multer, mysql2
- Фронтенд: Vanilla JS, HTML5, CSS3, Cropper.js
- База данных: MySQL

Особенности:
- При удалении чата старые сообщения не восстанавливаются
- При новом сообщении чат появляется в списке только с новыми сообщениями
- Вновь присоединившийся пользователь не видит старых сообщений
- Аватары обрезаются в квадрат перед загрузкой

Создай полный рабочий проект с чистым, современным дизайном и всей необходимой документацией.
```
